JavaScript 函數學習題目

1
Callback 基礎實作
建立一個名為 mathTool 的函數。它接受三個參數：num1, num2 以及一個 action（回呼函數）。

要求：呼叫 mathTool(10, 5, ...) 時，分別傳入不同的匿名函數來達成「相加」與「相減」。

範例輸出： 15, 5
function mathTool(num1, num2, action) { return action(num1, num2); } // 相加 console.log(mathTool(10, 5, (a, b) => a + b)); // 輸出: 15 // 相減 console.log(mathTool(10, 5, (a, b) => a - b)); // 輸出: 5

解析

Callback（回呼函數）是一種將函數作為參數傳遞的技術。mathTool 接收一個 action 函數作為參數，然後在內部執行這個函數。

這裡使用了箭頭函數語法，可以更簡潔地定義匿名函數。

2
匿名函數與立即執行 (IIFE)
寫出一個立即執行函數 (IIFE)，該函數內部定義一個區域變數 count = 100，並在執行時直接在控制台印出 "Count is: 100"。

要求：確保外部無法存取到這個 count 變數。
(function() { let count = 100; console.log("Count is: " + count); })(); // 外部無法存取 count - 會報錯 // console.log(count); // ReferenceError: count is not defined

解析

IIFE（Immediately Invoked Function Expression，立即呼叫函數表達式）是一種定義後立即執行的函數。

語法：(function() { ... })()

由於 count 是在函數內部宣告的區域變數，外部無法訪問，這就是閉包的作用之一，用於建立私有變數。

3
箭頭函數與陣列轉換
給定一個陣列 const prices = [100, 200, 300, 400]。請使用 map 方法結合箭頭函數，產生一個新陣列，內容是原價格打 8 折後的結果。

要求：程式碼需簡潔，盡量使用單行箭頭函數。
const prices = [100, 200, 300, 400]; // 方法1：完整箭頭函數 const discounted1 = prices.map(price => price * 0.8); // 方法2：更簡潔（隱含 return） const discounted2 = prices.map(p => p * 0.8); console.log(discounted1); // [80, 160, 240, 320] console.log(discounted2); // [80, 160, 240, 320]

解析

map() 方法會建立新陣列，陣列中的每個元素都是原始陣列中的每個元素呼叫一次所提供的函數後的結果。

單行箭頭函數的特色：當函數主體只有一個表達式時，可以省略大括號和 return 關鍵字。

4
陣列參數的「破壞性修改」
撰寫一個函數 cleanData(arr)。

內部邏輯：使用陣列方法移除傳進來的陣列最後一個元素，並在最前面加上 "Start"。

驗證：宣告 let myData = [1, 2, 3]，執行 cleanData(myData) 後，觀察 myData 的內容。
function cleanData(arr) { arr.pop(); // 移除最後一個元素 arr.unshift("Start"); // 在最前面加入 "Start" } let myData = [1, 2, 3]; cleanData(myData); console.log(myData); // ["Start", 1, 2]

解析

在 JavaScript 中，陣列是傳址呼叫（pass by reference）。這意味著當你將陣列傳入函數時，函數內部對陣列的修改會影響到原始陣列。

pop()：移除陣列最後一個元素，並返回該元素
unshift()：在陣列開頭加入新元素

注意：這就是所謂的「破壞性修改」，在開發中需要特別注意，避免意外修改原始資料。

5
函數回傳函數 (Higher-Order Function)
寫一個函數 multiplier(factor)，它會「回傳」另一個箭頭函數。這個被回傳的函數接受一個數字參數 n，並回傳 n * factor。

範例用法： 
const double = multiplier(2);
console.log(double(10)); // 預期輸出 20
function multiplier(factor) { return (n) => n * factor; } // 使用範例 const double = multiplier(2); const triple = multiplier(3); console.log(double(10)); // 20 console.log(triple(10)); // 30 console.log(multiplier(5)(10)); // 50

解析

Higher-Order Function（高階函數）是指接收函數作為參數，或回傳函數作為結果的函數。

multiplier 函數建立了閉包（closure）：內部函數可以存取外部函數的 factor 參數，即使外部函數已經執行完畢。

這是一個很強大的設計模式，可以用來建立函數工廠！

6
Callback 篩選器
手寫一個類似 filter 的函數，名為 myFilter(arr, callback)。

邏輯：建立一個新陣列，遍歷 arr，若 callback(item) 回傳為 true，則將該元素放入新陣列。

測試：使用此函數篩選出 [1, 5, 8, 12] 中大於 7 的數字。
function myFilter(arr, callback) { const result = []; for (let i = 0; i < arr.length; i++) { if (callback(arr[i])) { result.push(arr[i]); } } return result; } // 測試 const numbers = [1, 5, 8, 12]; const filtered = myFilter(numbers, n => n > 7); console.log(filtered); // [8, 12]

解析

myFilter 的實作展示了 callback 的經典用法：將判斷邏輯抽象化，讓函數更具彈性。

關鍵步驟：
1. 建立空陣列存放結果
2. 遍歷原始陣列的每個元素
3. 將 callback 函數套用在每個元素上
4. 若回傳 true，將該元素加入結果陣列

這就是 JavaScript 原生 Array.prototype.filter 的基本原理！

7
箭頭函數處理物件
給定一個包含物件的陣列 const users = [{name: "Alice", age: 25}, {name: "Bob", age: 17}]。

任務：使用 filter 與箭頭函數，篩選出所有 age 大於或等於 18 的人。
const users = [ {name: "Alice", age: 25}, {name: "Bob", age: 17} ]; const adults = users.filter(user => user.age >= 18); console.log(adults); // [{name: "Alice", age: 25}]

解析

當處理物件陣列時，箭頭函數的參數會是整個物件。我們可以透過 物件.屬性 來存取物件的特定屬性。

這裡 user => user.age >= 18 是箭頭函數的標準用法：
- user：代表陣列中的每個元素（物件）
- user.age：存取物件的 age 屬性

filter() 會回傳一個新陣列，包含所有符合條件的元素。

8
參數傳址陷阱：重新賦值 vs 修改
請問以下程式碼執行後，listA 與 listB 的內容分別是什麼？為什麼？
let listA = [1, 2]; let listB = [3, 4]; function process(a, b) { a.push(99); b = [100]; } process(listA, listB);
// 執行後： console.log(listA); // [1, 2, 99] - 被修改了！ console.log(listB); // [3, 4] - 維持原樣！

解析

關鍵概念：傳址 vs 重新賦值

listA 的情況：
a.push(99) 直接修改了陣列內容。由於 JavaScript 的物件是傳址，a 和 listA 指向同一個陣列，所以原陣列被修改了。

listB 的情況：
b = [100] 只是讓參數 b 指向一個新的陣列。這不會影響到外面的 listB，因為重新賦值並不會修改原變數的參照。

記憶口號：「修改」會影響外部，「重新賦值」不會！

9
延遲執行的 Callback
使用 setTimeout。要求在 2 秒後，利用箭頭函數印出陣列 ["Task", "Completed"] 中組合起來的字串 "Task Completed"。
const words = ["Task", "Completed"]; setTimeout(() => { console.log(words.join(" ")); // 2秒後輸出: Task Completed }, 2000);

解析

setTimeout 是非同步操作的經典範例：
- 第一個參數：callback 函數
- 第二個參數：延遲時間（毫秒）

join(" ") 會將陣列元素用指定的分隔符串聯成字串。

重要：setTimeout 不會阻塞程式執行，程式碼會立即繼續執行下面的內容，而 callback 會在指定的延遲時間後才被執行。

10
綜合應用：計算總價
寫一個函數 calculateTotal(cart, discountFunc)。

  • cart 是一個數字陣列（商品價格）
  • discountFunc 是一個處理折扣的 callback
任務：先將 cart 內所有數字相加，再將總和傳入 discountFunc 處理後回傳。

測試：傳入 [100, 200, 300] 並透過匿名函數扣除 50 元。
function calculateTotal(cart, discountFunc) { const sum = cart.reduce((acc, price) => acc + price, 0); return discountFunc(sum); } // 測試：傳入 [100, 200, 300] 並扣除 50 元 const total = calculateTotal( [100, 200, 300], (sum) => sum - 50 ); console.log(total); // 550

解析

這是一個綜合應用題，結合了多個概念：

1. reduce()：將陣列所有元素「歸納」成單一值（這裡是總和）
2. Callback：折扣邏輯由外部傳入，增加函數彈性
3. 箭頭函數：簡潔地傳入折扣邏輯 (sum) => sum - 50

這種模式讓 calculateTotal 函數可以被重複利用，只需更換不同的折扣 callback 就能達到不同的優惠方式！
